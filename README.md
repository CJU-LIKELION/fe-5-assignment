# 🦁 React 시작하기 (HTML 구조를 컴포넌트 단위로 쪼개기)

**아기사자 자기소개 페이지 UI를 React로 재구성하기**

이번 미션에서는 React라는 라이브러리를 통해 화면을 '컴포넌트(Component)' 단위로 쪼개어 조립하는 방식에 익숙해지는 것을 목표로 합니다. 기존의 단일 HTML 파일 방식에서 벗어나, 카드 한 장, 카드 목록 등 역할이 다른 부분을 각각 컴포넌트로 분리하고 `props`를 통해 데이터를 전달하며 화면을 렌더링하는 흐름을 경험하게 됩니다.

## 🎯 과제 목표

* React가 화면을 구성하는 방식이 왜 유용한지, **‘컴포넌트’라는 단위로 UI를 나누는 이유**가 무엇인지 설명할 수 있다.
* **JSX가 무엇인지**, HTML과 비슷해 보이지만 어떤 규칙과 특징을 가지는지를 설명할 수 있다.
* 기존 대시보드 UI를 역할 기준으로 분리하여, `Header`, `ProfileCard`, `List` 등과 같은 **컴포넌트로 재구성한 기준과 이유를 설명**할 수 있다.
* **`props`가 무엇인지**, `props`를 통해 전달한 값에 따라 컴포넌트가 어떻게 다르게 렌더링되는지를 예시를 들어 설명할 수 있다.
* 여러 컴포넌트로 구성된 화면에서 데이터가 어느 컴포넌트에서 시작해 어떤 경로로 흘러가는지 설명할 수 있다.

## 🛠 기술 스택

* **Language**: JavaScript (JSX), CSS
* **Framework**: React 18, React DOM
* **Build Tool**: Vite (Yarn 패키지 매니저 사용)
* **Environment**: Web Browser (Chrome 권장)

## 📋 주요 기능

1. **전체 화면 구성**: 웹 페이지는 하나의 화면으로 구성되며, 명단 조작 컨트롤 영역(상단), 요약 카드 영역(중단), 상세 정보 영역(하단)으로 나뉩니다.
2. **명단 조작 컨트롤 표시**: 기본 조작(추가/삭제 버튼, 총 인원) 영역과 외부 데이터 불러오기(랜덤 추가, 새로고침 등) 영역이 표시됩니다. **(실제 동작은 구현하지 않음)**
3. **보기 옵션 및 아기 사자 추가 폼 UI**: 필터/정렬/검색 UI 및 데이터 추가를 위한 폼 영역이 구성됩니다. 폼은 최종적으로 숨김 처리되며, **실제 동작은 구현하지 않습니다.**
4. **요약 카드 및 상세 정보 목록 렌더링**: 각 카드는 프로필 이미지, 배지, 이름, 파트 등의 정보를 포함하며, '내 카드'는 특별한 스타일로 구분됩니다. 그리드 형태 및 세로 목록 형태로 각각 렌더링됩니다.

## 🚀 실행 흐름 (Implementation Logic)

1. **프로젝트 실행**: Vite 기반의 React 프로젝트를 생성하고, `yarn install` 후 `yarn dev`로 개발 서버(기본 포트: 5173)를 실행합니다.
2. **스타일 유지**: 이전 주차에서 사용하던 CSS 파일을 수정 없이 그대로 사용하며, JSX의 클래스명은 CSS의 이름과 일치시킵니다.
3. **데이터 소스 사용**: `src/data/lions.js` 파일에 정의된 **mock 데이터**만을 사용하여 요약 카드, 상세 카드, 총 인원 정보를 렌더링합니다.
4. **데이터 기반 렌더링 (props 전달)**: 컴포넌트 간 데이터는 `props`를 통해 전달되며, 카드 목록 렌더링 시 각 항목에 `key` 속성을 적절히 지정합니다.
5. **디렉토리 구조**: `src` 폴더 내에 `main.jsx`, `App.jsx`, `data/lions.js`, `components/`(JSX 파일들), `styles/`(CSS 파일) 구조를 유지하며 컴포넌트를 분리합니다.

## ⚠️ 제약 사항 및 유의 사항

* **언어 / 실행 환경**: React, JavaScript(JSX)와 Vite 개발 서버를 사용하며, **TypeScript는 사용하지 않습니다.**
* **기능 범위**: UI 렌더링에만 집중하며, 버튼 클릭이나 폼 제출 등 **어떠한 상호작용 동작도 구현하지 않습니다.** 외부 API 요청(`fetch`) 및 React Hooks(`useState`, `useEffect`)는 사용하지 않습니다.
* **라이브러리 사용 범위**: React 18과 React DOM만 허용되며, CSS 프레임워크(Tailwind 등), 상태 관리(Redux 등), UI 라이브러리(Material-UI 등), 라우팅 라이브러리 등은 일절 사용하지 않습니다. 
* **스타일 작성 방식**: 이전 주차의 CSS 파일을 재사용하며, **새로운 스타일 추가나 수정은 금지**됩니다. CSS-in-JS나 인라인 스타일(`style`)도 사용하지 않습니다.
* **데이터 관리 방식**: 모든 데이터는 읽기 전용(`src/data/lions.js`)으로만 사용하며, 컴포넌트 내부에서 데이터를 직접 생성하거나 하드코딩하지 않습니다.
